# PDF 内容提取工具

这是一个用 Golang 编写的 PDF 内容提取工具，可以从 PDF 文档中提取结构化内容，包括标题、正文、页眉和页脚。

## 功能

- **标题提取**: 从 PDF 大纲（目录/书签）和文本内容中提取标题
  - 支持多种标题格式：'1. ', '1.1 ', '2.3.4 ', '2.3.4.' 等
  - 自动识别标题层级结构
  - 记录标题所在的页码
  
- **正文提取**: 提取 PDF 文档的主要文本内容
  - 自动排除标题、页眉和页脚
  - 保持段落结构和换行
  - 智能处理跨页文本连接
  
- **页眉识别**: 识别和提取页面顶部的重复性文本
  - 基于位置（页面顶部 15%）和跨页重复性识别
  - 支持奇偶页不同的页眉
  - 记录页眉出现的页码范围
  
- **页脚识别**: 识别和提取页面底部的重复性文本
  - 基于位置（页面底部 15%）和跨页重复性识别
  - 自动识别页码模式
  - 支持奇偶页不同的页脚
  - 记录页脚出现的页码范围

- **灵活的输出格式**:
  - 文本格式（默认）：清晰的分区显示
  - JSON 格式：便于程序处理和集成

```bash
go mod download
```

## 内容类型说明

### 标题 (Headings)

从 PDF 大纲（目录/书签）和文本内容中识别的标题。

- 支持多级标题格式：`1. `, `1.1 `, `2.3.4 `, `2.3.4.` 等
- 自动识别标题层级（根据点号数量）
- 记录标题所在的页码
- 大纲标题优先于文本标题
- 自动合并和去重

### 正文 (Body)

PDF 文档的主要文本内容。

- 自动排除标题、页眉和页脚
- 保持段落结构和换行
- 按页码组织内容
- 智能处理跨页文本连接
- 支持中英文混排

### 页眉 (Headers)

出现在每页顶部的重复性文本。

- 基于位置识别（页面顶部 15%）
- 基于跨页重复性识别（至少 3 个页面）
- 支持奇偶页不同的页眉
- 记录页眉出现的页码范围
- 自动合并相同的页眉

### 页脚 (Footers)

出现在每页底部的重复性文本。

- 基于位置识别（页面底部 15%）
- 基于跨页重复性识别（至少 3 个页面）
- 自动识别页码模式（支持多种格式）
- 支持奇偶页不同的页脚
- 记录页脚出现的页码范围
- 支持罗马数字页码

## 使用方法

### 基本用法

```bash
# 提取所有内容（标题、正文、页眉、页脚）
go run main.go document.pdf

# 或使用编译后的程序
./pdf-parser document.pdf
```

### 命令行参数

#### 基本选项

- `<PDF文件路径>` - 要解析的 PDF 文件路径（必需）
- `-h, --help` - 显示帮助信息并退出
- `-d, --debug` - 启用调试模式，显示详细的提取统计信息

#### 内容提取选项

- `--extract-all` - 提取所有内容类型（标题、正文、页眉、页脚）
  - 这是默认行为，如果未指定任何提取选项
- `--extract-body` - 仅提取正文内容
- `--extract-header` - 仅提取页眉内容
- `--extract-footer` - 仅提取页脚内容

**注意**: 可以组合多个提取选项，例如同时提取页眉和页脚

#### 输出格式选项

- `--format text` - 以文本格式输出（默认）
- `--format json` - 以 JSON 格式输出，便于程序处理

### 使用示例

```bash
# 提取所有内容（默认行为）
./pdf-parser document.pdf

# 仅提取正文内容
./pdf-parser --extract-body document.pdf

# 同时提取页眉和页脚
./pdf-parser --extract-header --extract-footer document.pdf

# 以 JSON 格式输出所有内容
./pdf-parser --format json document.pdf

# 启用调试模式查看提取统计
./pdf-parser -d document.pdf

# 组合多个选项：以 JSON 格式输出正文，并启用调试
./pdf-parser -d --extract-body --format json document.pdf
```

### 输出格式示例

#### 文本格式输出

```

级别 1: 第一章 引言 (第 1 页)

=== 正文 (第 2 页) ===
```

## 注意事项

### 内容提取方式

工具使用多种方式提取不同类型的内容：

#### 1. 标题提取

- **从 PDF 大纲提取**（主要方式）
  - 依赖 PDF 文件中的大纲（Outlines/Bookmarks）信息
  - 如果 PDF 没有大纲结构，将无法通过此方式提取标题

- **从文本内容提取**（辅助方式）
  - 识别符合特定格式的文本行作为标题
  - 支持的格式：`1. 标题`、`1.1 标题`、`2.3.4 标题`、`2.3.4. 标题` 等
  - 注意：unipdf 免费版本对文本提取有限制

#### 2. 正文提取

- 从每个 PDF 页面提取文本内容
- 自动排除标题、页眉和页脚
- 保持段落结构和换行
- 智能处理跨页文本连接

#### 3. 页眉/页脚识别

- **位置判断**: 基于文本在页面中的位置（顶部 15% 或底部 15%）
- **重复性检测**: 识别在多个页面重复出现的文本
- **页码识别**: 自动识别页脚中的页码模式
- 支持奇偶页不同的页眉/页脚

### 限制

- unipdf 库的免费版本可能有功能限制
- 商业用途需要购买 unipdf 许可证
- 某些 PDF 文件的内容可能无法被正确识别（取决于 PDF 的内部结构）
- 如果标题跨越多行或使用特殊格式，可能无法被识别
- 页眉/页脚识别依赖于位置和重复性，对于不规则布局可能不准确
- 多栏布局的正文提取可能需要特殊处理

### 解决方案

如果发现某些内容缺失或识别不准确：

1. **标题缺失**:
   - 检查 PDF 文件是否包含大纲（在 PDF 阅读器中查看目录/书签）
   - 如果大纲不完整，可能需要使用 PDF 编辑工具重新生成大纲
   - 确保标题格式符合支持的模式

2. **正文内容不完整**:
   - 检查是否被误识别为页眉或页脚
   - 使用 `--debug` 模式查看提取统计信息
   - 对于特殊布局的 PDF，可能需要调整分类阈值

3. **页眉/页脚识别不准确**:
   - 页眉/页脚需要在至少 3 个页面重复出现才会被识别
   - 位置阈值默认为 15%，可能需要根据具体 PDF 调整
   - 使用 `--extract-all` 查看完整提取结果以便调试
